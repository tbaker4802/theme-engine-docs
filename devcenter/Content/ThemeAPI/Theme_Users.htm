Users Endpoint

The users endpoint provides methods to create and update user accounts within the system and storefront. These user accounts typically are shoppers, with associated shipping addresses, billing addresses, and payment profiles per their ID. The endpoint also includes methods for verifying if an account already exists for the entered email and if the user is logged in currently.

This endpoint does not handle password resets. Use the Login endpoint for password changes.

A shopper may need to have an account when making purchases through your storefront. Depending on the settings of your storefront, guests may not need to create an account for completing purchases. When creating an account, you will perform the following steps:

  • Check if a user account exists with the GET method.
  • Create the user ID with the POST method.
  • Add or update data to the account using the appropriate methods for the shipping addresses, billing addresses, and payment profiles. The ID will be required for these calls.

The following information provides details on the request and the response status code and JSON with further data information.

User Account

Check for a User Account

Learn more about the GET method for verifying if a user account exists. When you perform this call, you receive a 200 status code if the account exists or a 404 status code if an account does not exist. When receiving a 404, you can then continue with a POST user method.

The URI call structure is:

POST http://{domain}/api/v1/users?email=email
Create a User

Learn more about the POST method for creating a user. When you perform this call, you receive a 201 status code and an ID. To further update and add content to the account, use the operations for shipping addresses, billing addresses, and payment profiles.

The URI call structure is:

POST http://{domain}/api/v1/users
Get Logged In User

Learn more about the GET method for retrieving user login status. When you perform this call, you receive a 200 status code and information if the account is actively logged in and by whom.

The URI call structure is:

GET http://{domain}/api/v1/users/current

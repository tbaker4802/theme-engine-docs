Get a Cart

This GET method requests a shopper's cart. The endpoint receives the cart ID from the cookie for the shopper. The response only provides a subset of cart objects: the cart item object and the cart totals object. Depending on the in-progress status of the shopper's experience, information returned on a GET includes NULL data. As the process moves from adding items to cart into the checkout stage, the data returns updates with more information available.

Most shopper specific data, such as the addresses and payment profile, depend on the shopper's ID. This is a unique ID associated to the shopper per their email address (their login account name).

You can also use this method to check the content of a cart for successful add to cart methods.

Request

The URI call structure is:

GET http://volusion.apiary-mock.com/api/v1/carts

Parameters

The request does not require parameters. The cart ID is received via a cookie.

Response

The response includes data for the cart item object and the cart totals object.

Data

The response includes data with detailed information per category and associated subcategories. Due to the parent-child relationship of categories, the data may return multiple nested subcategories with further nested subcategories.

The first set of attributes in data are a set of nested item attributes, one per product item in the cart when the method calls. Each object includes the following data:

Attribute Description Type
id

The generated system ID for the product.

 Number
code Alphanumeric product code for the product. This is the product code you entered when creating the product, not the generated product ID by the Volusion system. String
name The product name entered at product creation that describes the product. The name is alphanumeric and supports some special characters. This is a required setting and will always return content.

String
qty The numeric amount of the product item added to the cart. String
options

Nested attributes of options selected for the item in the cart.

  • id - The ID of the product option.
  • optionText - Optional. The text entered by the shopper when ordering the product. This String may include punctuation and formatting.
 String
kits Nested attributes of accessory items bundled together as a kit for the product in the cart. The kit includes data for each product associated to this product by code. String
taxable

This setting indicates if the product can be taxed, or is non-taxed. If false, taxes are not applied to this product during purchase. Non-taxed products vary state-to-state.

 True/False String
tax

This setting includes all rate amounts for the configured tax rates for the store:

  • tax1 - The base tax rate for the store.
  • tax2 - The Value Added Tax (VAT) tax rate for the store. This is the second tax rate you configure in the Admin.
  • tax3 - The Goods and Services Tax (GST, for Australian shoppers) tax rate for the store. This is the third tax rate you configure in the Admin.
 Number
imgUrl The URL location of the image file for the product in the cart. Example: //cdn3.volusion.com/ehn32.tcm2q/v/vspfiles/photos/13010440-1.jpg String
productUrl The URL location for the product in the cart. Example: /mobile/product.aspx?ProductCode=13010440 String
pricing

This data includes all pricing information for the product in the cart:

  • subtotal - Pre-taxed total amount of all products added to the cart.
  • recurringPrice - Provides information for any recurring price options for the product, such as with a subscription product. This price includes a starting price, normally charged price, and data on the length of term for charging each price.
    • price - The normally charged price for the recurring product that begins charging after the initial starting price completes its term. Example:
    • startPrice - Sets the initial price for the recurring charges, only charged during the starting duration.
    • thenXMonths - Sets the complete duration of the recurring price in months. This terms starts after the starting price term ends.
    • everyXMonths - Sets the recurring charge frequency in months.
    • startXMonths - Sets the complete duration of the starting price recurring charge. After this term, the normal duration begins.
  • unitPrice - The price of the specific product.
 Number
isGiftWrapAvailable This setting indicates if a gift wrap option is available for the product in the cart. If the shopper selects to add gift wrap to the product, they are charged the associated giftwrap price. True/False String
giftwrap

This setting includes additional information for gift wrap additions to the product:

  • price - The additional currency amount to add to the subtotal of the cart.
  • message - A String message entered by the shopper to include with the purchased product's gift wrap.
  • selected - Indicator if the gift wrap option is selected (true) or not included (false).
  
giftCardFromName The entered name of the person sending the gift card. Example: Mary Smith String
giftCardSendMethod The method for sending the gift card, such as email or postal mail. Example: email. String
giftCardToName The entered name of the person receiving the gift card. Example: Bill Mathis. String
giftCardEmailAddress The email address of the person receiving the gift card. Example: BillMathis@mail.com. String
giftCardMessage A String message to include in the email from the sender to the receiver for the gift card. Example: Happy Birthday, Bill! I hope you enjoy this gift. String

The second set of objects in data include the following:

Parameter Description Attribute
discounts This data includes nested discount attributes included or available for the cart. Depending on the selected products or entered codes by the shopper, this data may be empty or include numerous discounts. String
totals

The compiled totals for all product items and cart settings:

  • shipping - Total shipping charges.
  • tax2 - Total calculated VAT amount using the VAT rate (tax2) for the cart.
  • tax1 - Total calculated base tax amount using the base rate (tax1) for the cart.
  • kitItems - Total amount of kit items included with the product items.
  • discounts - Total amount of all discounts added to the cart.
  • tax3 - Total calculated GST amount using the GST rate (tax3) for the cart.
  • items - Total calculated currency amount of all product items in the cart.
  • giftWrap - Total calculated amount of all gift wrap options added to the cart.
  • qty - Total number of items in the cart.
  • grandTotal - Calculated total of the subtotal and tax amount added together, minus any discount and gift card amounts.
  • giftCardAmountUsed - Currency amount used from a gift card towards the grand total.
  • taxTotal - Total amount for all taxes calculated against the cart.

Number

Currency
shippingMethods Nested shipping method attributes for the cart. If the cart is still in-progress and not in the final order processing stage, this data may be empty.

Currently, shipping methods support three attributes (additional methods may be available in later releases):

  • qty - The total quantity of items added to the cart with gift wrapping added.
  • giftWrap selected - Indicates if gift wrapping is selected as true on the product. If false, gift wrap has not been selected.
  • giftWrap message - A String text message to add to the gift wrapping.
 String
shippingAddress

Data for the shipping address for sending all packaged products in the cart. If the cart is still in-progress and not in the processing stage of purchase, the shipping address may display with NULL values. This data is entered and available in the cart during those final transaction steps. The shipping and billing address can match.

  • city - The city for the address.
  • firstName - The first name of the recipient.
  • postalCode - A 5-9 digit postal code for address. This is also considered the zip code in the US.
  • address1 - The first line of the address.
  • address2 - The second line of the address.
  • lastName - The last name of the recipient.
  • country - The country location for the address.
  • companyName - Optional, the name of the company receiving the shipment.
  • id - The system ID for the shipping address.
  • residential - Indicates if the shipping address is a residential address.
  • phoneNumber - A 10 digit number to contact for the shopper receiving the shipment.
  • state - The state for the address.
  • preferred - Indicates if this shipping address is the preferred address. The preferred address is used for all shipping unless otherwise selected or entered.

String

True/False String
billingAddress

Data for the billing address, used for verification against the payment method. The billing and shipping address may match. If the cart is still in-progress and not in the processing stage of purchase, the billing address may display with NULL values. This data is entered and available in the cart during those final transaction steps.

  • country - The country for the shopper's billing address.
  • firstName - The first name of the shopper making the purchase.
  • city - The city the shopper's billing address.
  • postalCode - The postal code or zip code for the shopper's address.
  • phoneNumber - A 10 digit phone number for the shopper.
  • companyName - Optional, the shopper's company name.
  • id - The system generated ID for the billing object.
  • state - The state for the shopper's billing address.
  • address1 - The first line of the shopper's billing address.
  • lastName - The last name of the shopper making the purchase.
  • address2 - The second line of the shopper's billing address.
  • preferred - Indicates this is the preferred billing address, and should always be used for purchases unless otherwise selected.
 String
paymentProfile

This data provides the payment profile for the shopper, including the payment method they are using and all content entered to pass through the purchasing gateway or service you provide. PayPal data is included in another data set. If the cart is still in-progress and not in the processing stage of purchase, the payment profile may display with NULL values.

  • id - The system generated ID for the object.
  • method - The selected payment method for the purchase. Currently, only the credit card method is supported. Additional methods may be supported in later releases.
  • card - Data for the entered credit card to authenticate and process for the purchase.
    • holdersName - The full name of the card holder as it appears on the credit card.
    • expMonth - The expiration month in a numerical format.
    • issueMonth - The month the card was issued to the card holder.
    • issueNumber - The credit card number that was issued to the card holder.
    • expYear - The expiration year in a numerical format.
    • id - The system generated ID for the object.
    • last4 - Stores and displays the last four digits of the entered credit card number. This is used to inform the shopper of which card is being used, and for referencing the encrypted card number.
    • cardType - The credit card type for the profile.
    • cardNumber - The full credit card number for the account, typically 16 digits long.
    • pCIaaSId - The ID for the Payment Card Industry Data Security Standard (PCI) for regulating and ensuring all security for credit card purchases handled by the payment gateways through Volusion.
    • cardTypeName - The credit card type name, such as Visa or Mastercard.
    • issueYear - The year the card was issued to the card holder.
  • preferred - Indicates this payment method and profile is preferred, and should always be used for purchases unless otherwise indicated.

String

Number
paypalInfo

This data provides all PayPal payment processing information. PayPal must be an added payment option set through the Admin on the Settings > Payments page. If the cart is still in-progress and not in the processing stage of purchase, the payment profile may display with NULL values.

  • paymentStatus - The status of the payment through PayPal. PayPal may update and changed these status messages through their own code releases. The status may include the following:
    • Canceled_Reversal- A reversal has been canceled. For example, you won a dispute with the customer, and the funds for the transaction that was reversed have been returned to you.
    • Completed - The payment has been completed, and the funds have been added successfully to your account balance.
    • Created - A payment is made using Express Checkout.
    • Denied - You denied the payment.
    • Expired - This authorization has expired and cannot be captured.
    • Failed - The payment has failed. This happens only if the payment was made from your customerâ€™s bank account.
    • Pending - The payment is pending.
    • Refunded - You refunded the payment.
    • Reversed - A payment was reversed due to a chargeback or other type of reversal. The funds have been removed from your account balance and returned to the buyer.
    • Processed - A payment has been accepted.
    • Voided - This authorization has been voided.
  • totalAmount - The total amount of the purchase, to be charged and processed through PayPal.
  • transactionId - The transaction ID for referencing and handling the payment through PayPal processing.
  • paymentMethodId - The ID for the payment method. For PayPal, this is set to -1.
  • payerId - The ID for the payer, accepted and tracked through PayPal.
  • token - The identity token for the merchant's PayPal account.
  • useStoreCredit - Indicates if any store credit should be used towards the totalAmount of the purchase. This is set to false by default.

String

Number
customFields

Nested custom field attributes. If the custom fields have not been added to products or orders, the response is NULL. Custom fields are capable of storing additional data per product.

 String
customer

This data includes the shopper's initial important information needed for the cart. Additional shopper account information is available through the UPDATE for Carts (available attribute id) and User endpoints. The shipping address, billing address, and payment profile are associated to a customer.

  • id - The system generated ID for the shopper. Until the shopper logs in with an account or creates an account, they are considered a guest. As a guest, this ID is -1.
  • phoneNumber - The phone number for contacting the shopper directly.
  • pageName -
  • storeCredit - The amount of store credit accumulated and available for the shopper.
  • emailAddress - The email address associated to the shopper's account.
  
misc

This data includes all additional, or miscellaneous, content for the cart.

  • comment - The comments entered by the shopper for the cart. These comments may include questions or instructions for their order.
  • isGift - Indicates if the cart includes a product/item added with gift wrap included. This setting is NULL if gift wrapping is not available for your store and as default.
